# Telegram Network & System Administration Bot

Telegram-бот для удалённого администрирования серверов на базе Ubuntu. Позволяет управлять сервисами, отслеживать ресурсы, диагностировать сеть и создавать резервные копии конфигураций — прямо из мессенджера, без подключения по SSH.

Проект создан в рамках дипломной работы по теме **«Разработка чат-бота для автоматизации задач администрирования в Telegram»**.

## Возможности

- **Мониторинг ресурсов** — загрузка CPU, RAM и дискового пространства в реальном времени
- **Автоматический мониторинг** — фоновые отчёты каждые 5 минут (включается/выключается командой)
- **Управление сервисами** — проверка статуса и перезапуск служб через `systemctl`
- **Сетевая диагностика** — ping хостов и трассировка маршрутов
- **Резервное копирование** — архивация заданных файлов и конфигураций в `.tar.gz`, с отправкой архива в Telegram
- **Обновление пакетов** — запуск `apt update && apt upgrade`
- **Перезагрузка сервера** — удалённый `reboot`
- **Контроль доступа** — авторизация по Telegram ID и хешированному паролю, сессия действует 1 час

## Установка и запуск

**1. Клонировать репозиторий**

```bash
git clone https://github.com/satievaay/satieva4431networkbot.git
cd satieva4431networkbot
```

**2. Создать виртуальное окружение и установить зависимости**

```bash
python3 -m venv venv
source venv/bin/activate
pip install aiogram psutil python-dotenv
```

**3. Создать файл `.env`** (см. раздел ниже)

**4. Запустить бота**

```bash
python3 main.py
```

## Конфигурация `.env`

Создайте файл `.env` в корневой директории проекта. Ниже приведён пример с описанием каждой переменной:

```env
# Токен бота, полученный от @BotFather в Telegram
BOT_TOKEN=8017109866:AAGDQdG6dLNhQAn17CowN_EXAMPLE

# Список Telegram ID пользователей, которым разрешён доступ к боту (через запятую)
# Узнать свой ID можно у бота @userinfobot
ALLOWED_USER_IDS=310,6321

# ID чата/группы, в которой работает бот (нужен при использовании в групповом чате)
# Для личного чата можно не указывать
CHAT_ID=-46416084

# Пароль для авторизации в боте — хранится в виде MD5-хеша
# Сгенерировать: echo -n "ваш_пароль" | md5sum
BOT_PASSWORD=63a90eabb98050796b649

# Пути к файлам и директориям, которые войдут в резервную копию (через запятую)
BACKUP_FILES=/home/satievaay/.ssh/my_ssh_code,/etc/mysql/my.cnf

# Директория, куда сохраняются архивы резервных копий
BACKUP_DIR=/home/backup
```

## Команды бота

Перед использованием любой функциональной команды с непосредственным взаимодействием с сервером необходимо пройти авторизацию через `/auth`.

| Команда | Описание |
|---|---|
| `/start` | Приветствие и активация бота |
| `/auth <пароль>` | Авторизация. Сессия действует 1 час |
| `/status` | Текущая загрузка CPU, RAM и диска |
| `/update` | Обновление списка пакетов (`apt update`) |
| `/upgrade` | Обновление пакетов (`apt upgrade`) |
| `/restart_service <имя>` | Перезапуск указанного сервиса, например `/restart_service nginx` |
| `/reboot` | Перезагрузка сервера |
| `/ping <хост>` | Проверка доступности узла, например `/ping google.com` |
| `/traceroute <хост>` | Трассировка маршрута до хоста |
| `/backup` | Создание архива файлов из `BACKUP_FILES` и его отправка в Telegram |
| `/monitor_start` | Включить автоматический мониторинг (отчёт каждые 5 минут) |
| `/monitor_stop` | Отключить автоматический мониторинг |

## Автозапуск через systemd

Чтобы бот запускался автоматически после перезагрузки сервера, создайте unit-файл:

```bash
sudo nano /etc/systemd/system/tgbot.service
```

Содержимое файла:

```ini
[Unit]
Description=Telegram Administration Bot
After=network.target

[Service]
User=your_username
WorkingDirectory=/path/to/satieva4431networkbot
EnvironmentFile=/path/to/satieva4431networkbot/.env
ExecStart=/path/to/satieva4431networkbot/venv/bin/python3 main.py
Restart=always

[Install]
WantedBy=multi-user.target
```

Затем включите и запустите:

```bash
sudo systemctl daemon-reload
sudo systemctl enable tgbot --now
sudo systemctl status tgbot
```

## Безопасность

- Доступ ограничен по списку `ALLOWED_USER_IDS` - бот игнорирует все сообщения от посторонних
- Авторизация через хешированный пароль (MD5); сессия автоматически истекает через 1 час
- Пароль и токен хранятся исключительно в `.env`, который исключён из репозитория через `.gitignore`
- Системные команды выполняются с ограниченными привилегиями `sudo`; права бота настраиваются через `/etc/sudoers`
