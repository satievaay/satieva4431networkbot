# Telegram Bot for Network & System Administration
*Данный бот предназначен для удаленного управления сервером на базе Ubuntu и мониторинга сетевой инфраструктуры. Бот позволяет администратору выполнять критические задачи через интерфейс Telegram, ускоряя реакцию на инциденты.*

**Основные функции:**
- *Мониторинг ресурсов: получение данных о загрузке ЦП, ОЗУ и дискового пространства в реальном времени;*
- *Сетевая диагностика: выполнение команд ping и проверка доступности узлов;*
- *Автоматические уведомления: фоновый мониторинг системы с оповещением каждые 5 минут;*
- *Безопасность: многоуровневая авторизация по Telegram ID и хеширование паролей (SHA-256);*
- *Управление процессами: просмотр активных процессов и возможность перезагрузки служб.*

**Технологический стек:**

- Язык: Python 3.10+

- Библиотека: aiogram 3.x (Asynchronous Framework)

- Мониторинг: psutil

- База данных/Конфиг: SQLite / .env

- Среда развертывания: Ubuntu Server
	
**Быстрое развертывание:**
1. Клонирование репозитория

`
	git clone https://github.com/satievaay/satieva4431networkbot.git
`

`
	cd satieva4431networkbot
`

2. Настройка окружения

`
	python3 -m venv venv
`

`
	source venv/bin/activate
`

`
	pip install -r requirements.txt
`

3. Конфигурация

Создайте файл .env в корневой директории и добавьте ваши данные:
```
BOT_TOKEN=your_telegram_bot_token
ADMIN_ID=your_telegram_id
BOT_PASSWORD=your_hashed_password
BACKUP_FILES=/your/path/1, your/path/2, your/path/2/file
BACKUP_DIR=/your/path/to/backup_dir
```

*Примечание: в случае использования бота в групповых чатах необходимо также добавить следующую строку:*

```
CHAT_ID=your_telegram_chat_id
```

4. Запуск через Systemd (рекомендуется)

Для обеспечения работы 24/7 создайте unit-файл в `/etc/systemd/system/tgbot.service`:

``` sudo systemctl enable tgbot --now ```

**Сетевые интерфейсы и протоколы передачи данных**

*Взаимодействие компонентов системы реализовано в рамках стандартного стека TCP/IP.*

| Компонент | Протокол / Технология | Описание |
| :--- | :--- | :--- |
| **Data Transport** | **TCP** | Обеспечение целостности данных при передаче команд и отчетов. |
| **Security Layer** | **TLS 1.3 / SSL** | Криптографическая защита трафика между сервером и шлюзом Telegram. |
| **Network Control** | **ICMP** | Использование эхо-запросов для анализа доступности сетевых узлов. |
| **API Gateway** | **HTTPS (Port 443)** | Прикладной интерфейс для взаимодействия с распределенными системами. |
* *Управляющие данные упаковываются в HTTPS-запросы, обеспечивая прохождение через файрволы без дополнительной настройки портов.*
* *Реализован прямой доступ к сетевому стеку ОС для мониторинга состояния интерфейсов и анализа сетевых задержек.*
